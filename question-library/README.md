# Question Library

질문 유형별 레퍼런스를 모아 두는 폴더입니다.

회사별 `references/`와 달리, 이 폴더는 특정 회사가 아니라 질문 의도 기준으로 사례를 정리할 때 사용합니다.

## 이 폴더를 쓰는 목적

- 비슷한 질문을 한곳에서 모아 본다.
- 원문만 저장하지 않고 구조 분석을 함께 남긴다.
- 반복해서 보이는 좋은 문단 구조와 전개 방식을 찾는다.
- 여러 사례에서 다시 확인된 패턴만 `references/few-shot.md`나 공통 규칙 문서로 올린다.

## 질문 유형 폴더

- `motivation`
- `job-fit-and-career-vision`
- `problem-solving`
- `collaboration`
- `communication-and-conflict`
- `strengths-and-weaknesses`
- `achievement-and-challenge`
- `education-and-learning`
- `values-and-criteria`
- `adaptability-and-attitude`

## 분류 기준

- 기준은 질문 문구가 아니라 평가 의도입니다.
- 한 항목은 대표 유형 1개에만 넣습니다.
- 여러 의도가 섞인 문항은 보조 태그로만 남깁니다.

예시:

- 지원동기 중심 질문 -> `motivation`
- 문제 해결 경험 중심 질문 -> `problem-solving`
- 갈등 해결과 커뮤니케이션 중심 질문 -> `communication-and-conflict`

## 항목에 들어갈 내용

각 항목에는 아래 내용을 남깁니다.

- 질문 원문
- 답변 원문
- 출처 링크
- 회사명과 직무명
- 대표 질문 유형
- 보조 태그
- 문단 수
- 문단 순서와 역할
- 핵심 경험 수
- 느낀점과 해석 위치
- 잘 쓴 이유
- 현재 스킬에 반영할 포인트

## 어떻게 활용하는지

이 폴더는 실제 작성 전에 아래 상황에서 참고합니다.

- 문항 의도를 어떻게 해석할지 애매할 때
- 문단 구조를 어떻게 잡아야 할지 막힐 때
- 비슷한 질문에서 경험을 어떤 방식으로 배치했는지 보고 싶을 때

우선순위는 항상 아래 순서입니다.

1. 현재 회사의 `company.md`
2. 현재 회사의 `references/`
3. `question-library/`

즉, 이 폴더는 회사별 자료를 대신하는 것이 아니라 보조 기준으로 씁니다.

## 운영 원칙

- 단일 사례를 바로 공통 규칙으로 올리지 않습니다.
- 같은 구조와 전략이 여러 사례에서 반복 확인될 때만 `references/few-shot.md`나 공통 규칙 문서로 승격합니다.
- 원문 레퍼런스는 저작권 이슈가 생길 수 있으므로 로컬 작업 데이터로 취급하는 것이 안전합니다.
